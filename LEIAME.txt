#
# Instalando e rodando o Hugo
#

O tema que estamos usando exige uma versão muito recente do Hugo. Recomendo
baixar o executável (prebuilt-binary) e salvá-lo na raiz do repositório.
https://gohugo.io/installation/

Para testar o site localmente, rode "./hugo serve" e abra o seu web-browser
em https://localhost:1313/python-ufrj. Ele atualiza o browser automático
quando você salva o arquivo com o código fonte.

Para criar uma nova postagem, use "./hugo new content/aulas/aula-NN.md".
Isso irá criar um arquivo vazio com o cabeçalho para você preencher.
Também pode copiar e modificar o arquivo de uma aula existenteo.

#
# Lidando com URLs relativos e absolutos
#

Hugo (o software) não lida bem com baseURL que termina em diretório. Os
URLs absolutos são relativos à raiz do domínio, em vez de relativos ao
baseURL. Por exemplo, um "/images" vai pra "github.io/images" em vez de
"github.io/python-ufrj/images" como gostaríamos.

1. Em templates HTML, use a função absURL para calcular o URL.
   Importante: não pode ter "/" no começo da string! (bug do Hugo)
   Você pode encontrar exemplos na pasta layout.

2. Dentro de postagens markdown, sempre use URLs relativos.
   Não dá pra usar absURL dentro do markdown. Se preciso, crie pastas dentro
   dentro do static/ que espelham a hierarque do content/
   Um exemplo são as figuras de cada aula.
